Aba 13: @starting-style + transition-behavior

Animar propriedades "discretas" como display, visibility e overlay em transitions. Requer transition-behavior: allow-discrete e bloco @starting-style. Chrome 117+

Modal com @starting-style

.modal {
  opacity: 0;
  display: none;
  transition: opacity 300ms ease,
              display 300ms allow-discrete;
}
@starting-style {
  .modal { opacity: 0; }
}
input:checked ~ .modal {
  opacity: 1;
  display: flex;
}

Tooltip Hover

↗ Tooltip que aparece suavemente
.tooltip {
  opacity: 0;
  transform: translateY(-8px);
  transition: opacity 250ms ease,
              transform 250ms ease;
}
.trigger:hover + .tooltip {
  opacity: 1;
  transform: translateY(0);
}
@starting-style {
  .tooltip { opacity: 0; transform: translateY(-8px); }
}

Conteúdo com max-height animado. O @starting-style define o estado inicial da entrada.

details .content {
  max-height: 0;
  opacity: 0;
  transition: max-height 350ms ease,
              opacity 250ms ease;
}
details[open] .content {
  max-height: 200px;
  opacity: 1;
}
@starting-style {
  details .content { opacity: 0; }
}

Por que @starting-style?

Sem @starting-style, transições em display não animam a entrada do elemento — apenas a saída. O bloco @starting-style define o estado "de onde vem" a transição.

/* SEM @starting-style: */
.card { display: none; transition: opacity 300ms; }
.card.show { display: block; opacity: 1; }
/* Problema: a opacity transition
   não roda na entrada (display: none → block)
   porque o browser não tem o valor inicial */

/* COM @starting-style: */
@starting-style {
  .card.show { opacity: 0; }
}
/* Agora o browser sabe: o opacity
   começa em 0 e vai até 1 */
