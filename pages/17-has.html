Aba 17: :has() — Parent Selectors Animados

:has() permite selecionar um elemento baseado em seus filhos. Combinar com @property e transitions permite criar UIs reativas sem JavaScript. Chrome 105+

Form Validation Shake

Email válido
.field:has(input:invalid:not(:placeholder-shown)) {
  animation: shake 400ms ease;
}
@keyframes shake {
  0%, 100% { transform: translateX(0); }
  20%, 60% { transform: translateX(-6px); }
  40%, 80% { transform: translateX(6px); }
}

Card Checkbox Selecionado

.card {
  border: 2px solid var(--border);
  transition: border-color 300ms,
              transform 300ms;
}
.card:has(input:checked) {
  border-color: var(--accent);
  transform: scale(1.02);
}

Grid: Destacar Item Hovered

/* Hover em qualquer filho afeta
   todos os outros */
.grid:has(.cell:hover) .cell {
  opacity: 0.4;
  transition: opacity 250ms;
}
.grid:has(.cell:hover) .cell:hover {
  opacity: 1;
  background: var(--accent);
  transform: scale(1.1);
}

Section Focada

.section {
  border: 2px solid var(--border);
  transition: border-color 300ms,
              background 300ms;
}
.section:has(input:focus) {
  border-color: var(--accent);
  background: rgba(255, 255, 255, .05);
}

Counter via :has()

@property --count {
  syntax: '<integer>';
  initial-value: 0;
  inherits: true;
}
.counter {
  --count: 0;
  counter-reset: c var(--count);
}
.counter:has(input:checked:nth-of-type(1)) { --count: 1; }
.counter:has(input:checked:nth-of-type(2)) { --count: 2; }
/* etc */

Suporte e Limitações

Suporte: Chrome 105+, Safari 15.4+, Firefox 121+ (set/2023), Edge 105+

Limitações: 

/* Não funciona com :has() aninhado
   profundo em alguns browsers */
.parent:has(.child:has(.grandchild)) /* ⚠️ */

/* Não pode usar em pseudo-elements
   como ::before, ::after */
.elem::before:has(...) /* ❌ */

/* Performance: pode ser custoso
   em selectors complexos em
   árvores grandes */
